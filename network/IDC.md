#### 인덱스


## IDC 역할

IDC: Internet Data Center

서버, 스토리지, 네트워크 장비를 24시간 365일 안정적으로 운영하기 위해 모든 인프라를 집약해둔 공간을 IDC라고 한다

IDC 목적
- 항상 켜져 있어야 하는 서버를 안전하게 유지한다
- 빠르고 안정적인 네트워크를 제공한다
- 장애/재해 상황에서도 서비스 지속성을 보장한다


## IDC 구성요소

### IT 장비

서버: 데이터를 처리하는 고성능 컴퓨터 (WAS, DB 등)

스토리지: 대용량 데이터 저장 장치

네트워크: 코어 스위치, 라우터, 방화벽, ISP 회선 (KT, SKB, LGU+ 등)

### 기반 시설

이중 전원 공급

UPS (무정전 전원 장치)

비상 발전기/배터리실

항온항습 (에어컨 및 냉방 시스템)

### 운영/유지 시스템

운영 및 관제 시스템

통신 네트워크

보안 시스템


## IDC 서비스 형태

**코로케이션 (Colocation)**
- 서버는 기업 또는 개인이 소유
- IDC는 공간, 전원, 네트워크만 제공하는 방식

**서버 호스팅**
- IDC가 서버까지 제공
- 고객은 OS/애플리케이션만 운영하는 방식

**클라우드**
- IaaS(Infrastructure as a Service): 데이터센터를 통해 서버와 스토리지 등 컴퓨팅 자원을 제공하는 서비스
- PasS(Platform as a Service): 데이터센터를 통해 소프트웨어 서비스를 개발할 때 필요한 플랫폼을 제공하는 서비스
- SaaS(Software as a Service): 클라우드 환경에서 데이터센터를 통해 운영되는 애플리케이션 서비스


## IDC vs 클라우드

IDC: 서버를 빌리는 공간 (운영의 책임이 고객에게 있음)

클라우드: 서버를 서비스로 쓰는 것 (운영의 책임을 공급자에게 위임)

### 구조적 차이

IDC
- 실제 서버, 랙, 케이블 존재
- 네트워크 구성도 직접 설계

```text
서버 - 스위치 - 방화벽 - IDC 네트워크
```

클라우드
- 서버는 API 호출로 생성
- 논리적 네트워크 (VPC)
- 유연한 스케일링

```text
EC2 생성 -> 서브넷 할당 -> 보안그룹 연결
```

### 비용 차이

IDC
- 초기 비용이 큼
- 고정 비용 구조로 트래픽이 많거나 적어도 비용이 동일함

클라우드
- 초기 비용이 작음
- 사용량 기반 과금 구조로 트래픽이 늘면 바로 비용이 증가함
